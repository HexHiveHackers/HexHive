# MP2K / SMF feature audit — `/sounds/midi-lab` fixtures

Generated by `bun run scripts/audit-midi-features.ts`. Reports every event class our parser
sees in each fixture .mid, plus a scan of the paired `.s` for Sappy bytecode commands. The
"Verdict" column is a quick read on whether the feature is handled by our playback (rewriter
+ spessasynth) or being silently dropped.

## Executive summary

Three categories of finding, in order of audible impact:

### A. Sappy state encoded in non-standard CCs (rewriter handles)

The .s → .mid converter preserves Sappy state by emitting it as **non-standard MIDI controllers (CC20-CC30 range)**. spessasynth doesn't interpret them directly, so the rewriter (`src/lib/midi-lab/midi-rewrite.ts`) translates each into the standard MIDI equivalent the synth honors. Mappings empirically confirmed by exact count + value match against the .s sources across all three fixtures:

| CC | Sappy command | What it controls | Rewriter target | Status |
|---|---|---|---|---|
| **20** | `BENDR` | Pitch-bend range in semitones (most fixtures use 12 = ±1 octave) | RPN 0,0 + CC6 (pitch-bend range) | ✅ shipped |
| **21** | `LFOS` | LFO speed (vibrato rate) | CC76 (XG vibrato rate) | ✅ shipped |
| **29** | `XCMD xIECV` | Echo / initial channel volume | CC91 (reverb send) — best-effort, spessasynth reverb character ≠ GBA echo | ✅ shipped |
| **30** | (unknown XCMD subcommand, constant value 8) | TBD — likely Sappy-specific | — | ⚠️ unhandled; low impact |

### B. Loop boundaries present but not consumed

All three fixtures carry SMF Marker events with text `[` (loop start) and `]` (loop end) — the canonical Sappy/m4a loop-point convention. **Our playback ignores them**: `seq.loopCount = ∞` simply restarts the whole SMF from frame 0, so any pre-loop intro repeats every time.

**Fix sketch:** before passing the SMF to spessasynth's Sequencer, rewrite `[`/`]` markers into the `loopStart`/`loopEnd` text spessasynth's built-in marker loop handler recognizes — or call spessasynth's loop-point API directly with the tick values.

### C. Inherent limits (no fix without a custom synth)

- PSG / DMG square-wave + noise voices: SF2 sample-based playback can only approximate, not reproduce.
- Pulse-wave duty cycle shifts (12.5/25/50/75%): no SF2 modelling.
- Voice stealing / hard polyphony cap: spessasynth has unbounded voices, so busy passages render fuller than GBA hardware would.
- `MODT 1`/`MODT 2` (volume / pan vibrato): dropped during .s → .mid (no SMF representation).

These would need a dedicated GBA-engine synth (e.g. agbplay's WebAssembly port) to fix. Not in scope.

| Pulse-wave duty cycle | no SF2 equivalent | ❌ |
| MODT 1/2 (volume/pan vibrato) | lost in .s → .mid | ❌ |
| Sappy TIE | flattened to long NoteOn/Off — works | ✅ |
| Voice stealing / hard polyphony cap | spessasynth has unbounded polyphony | ❌ over-renders busy passages |

- ✅ **Sappy CC20 (BENDR) → RPN 0,0 + CC6** — rewriter restores intended pitch-bend range. (Shipped.)
